# EDR v0.8 变更记录

> 日期: 2026-06-25 | 基于: v0.7.6

## 概述

v0.8 聚焦安全强化，四方向同步推进：管理认证、策略硬化、CLI 研判、自保护加固。

---

## 一、管理认证：Admin Token

### 新增文件

| 文件 | 说明 |
|------|------|
| `internal/adminauth/adminauth.go` | HMAC-SHA256 令牌签发/验证 |
| `internal/adminauth/adminauth_test.go` | 8 个测试用例 |

### 新增 API

| 端点 | 方法 | 说明 |
|------|------|------|
| `POST /v0/admin/token` | 令牌签发 | 申请指定操作令牌，5 分钟有效 |
| `POST /v0/admin/restart` | 令牌验证 | admin token 授权代理重启 |

### 修改文件

| 文件 | 变更 |
|------|------|
| `internal/control/server.go` | ServerOptions 新增 AdminKey/Restart 字段；shutdown handler 增加 admin token 优先路径；新增 `/v0/admin/token`、`/v0/admin/restart` 端点 |
| `internal/control/security.go` | 新增 `requireAdminAuth()`、`adminAuthOK()` 安全校验函数 |
| `internal/runtime/config.go` | 新增 `AdminAuthConfig` 结构体 (`admin_key_path`)，Config 新增 `AdminAuth` 字段 |
| `cmd/edr-agent/main.go` | 启动时加载 admin key；新增 `requestRestart` 闭包 + `execSelf()` 函数 |
| `cmd/edrctl/main.go` | 新增 `admin gen-key/token/shutdown/restart` 子命令 + `unixPostRawAuth()` 辅助函数 |

## 三、CLI 研判工作台

### 新增命令

```bash
edrctl investigate <event_id>              # 五面板事件研判
edrctl pstree [--detail] [--filter=S]      # 进程树可视化
edrctl audit export [from=][to=][format=]  # SIEM 兼容导出
edrctl audit integrity                      # 完整性报告
edrctl audit timeline [from=][to=]          # 取证时间线
```

### 修改文件

| 文件 | 变更 |
|------|------|
| `cmd/edrctl/main.go` | 新增 investigate 命令组（fetchEvent/subjectPID/printRuleMatch/printBehaviorTimeline/printResponseRecords/printNetworkConnections/printFileOperations）；pstree 命令组（printProcTree/sortInts/toFloat）；audit 命令组（printCEF/printLEEF/printIntegrityReport/sevMap）；admin 命令组（上文） |
| `internal/control/server.go` | eventQuery 新增 EventID/SubjectPID 字段；`GET /v0/events` 支持 `event_id`/`subject_pid` 过滤参数 |

## 四、自保护加固

### BPF 探针变更

| 文件 | 变更 |
|------|------|
| `internal/bpf/probes/selfprotect.bpf.c` | 新增 `handle_process_vm_writev` kprobe；新增 `should_kill_caller()` 函数（SIGKILL/SIGHUP/SIGINT/SIGQUIT 总是杀，SIGTERM 仅当 caller ≠ PID 1 时杀）；handle_kill/handle_tgkill/handle_pidfd_send_signal 全部改用 `should_kill_caller` |
| `internal/bpf/probes/bpf_guard.bpf.c` | 新增 `handle_init_module` kprobe；新增 `handle_delete_module` kprobe；扩展 bpf_guard 拦截 BPF_PROG_DETACH（原仅 BPF_MAP_UPDATE_ELEM） |
| `internal/bpf/probes/connect.bpf.c` | 新增 ring0 网络黑名单检查：读取 `net_blacklist_enabled/ip/port` map → 命中即 `bpf_send_signal(9)` |
| `internal/bpf/probes/common.bpf.h` | 新增 `net_blacklist_enabled`(ARRAY)、`net_blacklist_ip`(HASH)、`net_blacklist_port`(HASH) 三个 BPF map |
| `internal/bpf/loader.go` | MapFiller 接口新增 SetNetBlacklistEnabled/NetBlacklistIPAdd/IPClear/PortAdd/PortClear |
| `internal/bpf/loader_libbpf.go` | CGo 实现：新增 map fd 字段、C 辅助函数、Go 包装器、`parseIPv4ToBE()` |
| `internal/bpf/fake.go` | FakeLoader 新增 5 个网络黑名单 stub 方法 |

### Fanotify 变更

| 文件 | 变更 |
|------|------|
| `internal/fanotify/fanotify.go` | Provider 新增 protectedInodes 集合 + ProtectFile/markFile/isProtectedInode 方法；handleEvent 在 handler 调用前注入 inode 保护检查；handleEvent 在 bypass 检查前注入 shell TTY 检测（isShell + hasTTY ± hasTTY 跳过 bypass 直接走策略评估） |

### Agent 变更

|------|------|
| `cmd/edr-agent/main.go` | 启动时调用 fanotify.ProtectFile 注册 6 个受保护文件（含 /proc/<pid>/mem）；新增 guardian 双守护模块（`startGuardian` ForkExec + heartbeat goroutine + 10s 守护检查）；新增 `--guardian` 内部 flag |
| `systemd/edr-agent.service` | ExecStop=/bin/true; KillMode=none; SendSIGKILL=no; TimeoutStopSec=1 |

### Rootkit 检测变更

| 文件 | 变更 |
|------|------|
| `internal/rootkit/detector.go` | RunOnce 新增 DetectHiddenConnections() + CheckSyscallIntegrity() 调用；新增 DetectHiddenConnections/readProcNetAddrs/hexToIP 函数；新增 CheckSyscallIntegrity 函数 |
| `internal/collector/merge.go` | MergedCollector 新增 seenAddrs 字段 + SeenAddrs() 方法；applyConnect 新增地址记录逻辑 |
