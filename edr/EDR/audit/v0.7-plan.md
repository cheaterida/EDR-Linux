# EDR v0.7 规划：运维审计可用性提升

> 日期: 2026-06-16 | 状态: 已实施（含 rootkit 检测插队项）| 基于: DESIGN.md + v0.6-decision-record.md + PROJECT_STATUS.md + v0.7-rootkit-decision-record.md

## Context

v0.6 检测能力已足够（89 规则、10 探针），瓶颈在蓝队操作效率。

**现状审计**：事件搜索 API（`GET /v0/events` + `queryEvents`）及其 `edrctl events query` 对接已实现，支持 category/severity/rule_id/file_path/subject/since/until/分页。本次仅需填补小的过滤缺口，重心放在 `edrctl report generate` 和同源归并。

**原则**：不做 Web 面板。所有审计操作通过 `edrctl` CLI 完成。废弃但未清理的旧 Web 代码一并移除。

### 1. edrctl report generate — 赛后自动报告

**命令**：
```bash
edrctl report generate \
  --from "2026-06-16 00:00" \
  --to   "2026-06-16 23:59" \
  --output report.json
```

**报告内容**：
- 总览：总事件数、告警数、阻断数、触发规则数
- 按主机分组：各主机告警/阻断统计 + Top 规则
- 按规则分组：各规则命中次数 + 涉及主机
- 攻击时间线：按时间排列的告警事件链
- 攻击链还原：利用 ProcTree 将同源事件关联为攻击阶段
- 响应记录：所有 kill/quarantine/isolate 的执行结果
- 完整性：日志链验证状态

**实现**：
- 后端新增 `POST /v0/report/generate` 端点，复用现有 `queryEvents` 扫描 + ProcTree 关联
- `edrctl` 新增 `report generate` 子命令

**关键文件**：
- `internal/control/report.go` — 新增：报告聚合 + 攻击链关联逻辑
- `internal/control/server.go` — 新增 `/v0/report/generate` handler
- `cmd/edrctl/main.go` — 新增 `report generate` 子命令

### 2. 同源事件自动归并

**目标**：同一 PID 在短时间内的多条告警合并为一条聚合通知。

**归并规则**：
- 同一 PID + 不同 rule_id 在可配窗口（默认 5s）内连续触发 → 合并为一条 `aggregated_alert`
- 聚合事件含 `merged_count`、`trigger_rules` 列表、`first_seen`/`last_seen`
- Webhook 输出中体现 `merged: true`

**实现**：
- 在现有 Suppressor 旁新增 Merger，维护 per-PID 最近事件环形缓冲
- 事件管线在 emit 前过 Merger

**关键文件**：
- `internal/control/suppress.go` — 扩展或新增归并窗口
- `internal/control/agent.go` — 事件管线集成

### 3. 事件查询过滤缺口补齐

已有参数：category、severity、rule_id、file_path、file_op、subject_name、subject_path、subject_cmdline、since、until、limit、offset

**新增**：
- `host` — 按主机名过滤（当前 event 中有 host 字段但未暴露过滤）
- `decision` — 按 allow/alert/block 过滤
- `&format=summary` — 可选：仅返回时间、主机、规则、严重级别、决策、动作的紧凑摘要，方便 CLI 表格展示

**关键文件**：
- `internal/control/server.go` — eventQuery 结构体 + eventQueryFromRequest 新增字段

### 4. edrctl 输出增强

- `edrctl events query` 结果默认以紧凑表格输出（非 raw JSON），加 `--json` flag 恢复原格式
- `edrctl status` 输出增加进程树节点数、连接追踪器状态、最近阻断记录数

**关键文件**：
- `cmd/edrctl/main.go` — 表格输出格式化
